# LLaDA2.1: Speeding Up Text Diffusion via Token Editing — 论文摘要

> **论文标题**: LLaDA2.1: Speeding Up Text Diffusion via Token Editing
> **arXiv ID**: 2602.08676
> **作者**: Ant Group 等
> **发布日期**: 2026-02-09
> **模型规模**: LLaDA2.1-Mini (16B), LLaDA2.1-Flash (100B)

---

## 1. 背景与动机

离散扩散语言模型（discrete diffusion Large Language Models, dLLMs）近年来作为自回归（AR）生成的重要替代方案迅速崛起，其核心优势在于支持非单调推理和并行解码。然而，标准的吸收态（absorbing-state）框架存在一个根本性的局限：它强制要求从 [MASK] 到固定 token 的单调、单向转换，这导致生成质量在并行解码时容易出现 token 级别的不一致性，且一旦出错难以修正。

LLaDA2.0 已经展示了 100B 级别块扩散模型的规模化潜力及其固有的并行化优势，但在**解码速度**与**生成质量**之间的微妙平衡始终未能被突破。LLaDA2.1 正是为了超越这一权衡而诞生的范式革新——它不再单纯追求参数量或刷榜分数，而是通过引入**可编辑性（editability）**来重新定义 dLLM 的推理范式。

---

## 2. 问题定义

本论文试图解决的核心问题是：

> **如何在保持甚至提升生成质量的前提下，显著加速 dLLM 的推理速度？**

传统的 dLLM 解码受限于吸收态框架的刚性结构：token 一旦被解码为确定性值，就无法在后续步骤中被修改。这种“一锤子买卖”的机制在并行解码场景下尤为致命——独立采样的 token 之间缺乏协调性，局部错误会迅速累积并放大（即 Exposure Bias），导致模型在后续步骤中变得越来越保守，生成速度急剧下降。虽然已有工作尝试通过置信度重掩码（confidence-based remasking）或外部引导模型来缓解这一问题，但均未能从根本上改变模型只能“生成”不能“修改”的困局。

## 3. 核心创新

LLaDA2.1 提出了以下几个关键创新：

### 3.1 可编辑状态演化（Editable State Evolution）框架

这是论文最核心的概念创新。与吸收态模型不同，LLaDA2.1 的状态转移不限于 `[MASK] → token`，而是同时支持 `token → token` 的编辑操作。该框架通过**双重概率阈值**机制（dual probability thresholds）控制解码动态：

- **解掩码阈值** $\tau_{\text{mask}}$：控制 [MASK] 位置何时被解码为具体 token。
- **编辑阈值** $\tau_{\text{edit}}$：控制已生成 token 何时被替换为更优候选。

在每个时间步 $t$，模型维护两个活跃更新集合：

- **解掩码集合** $\Gamma_t = \{ i \mid x_t^i = \text{[MASK]}$ 且 $p_\theta(v_t^i \mid \bm{x}_t) > \tau_{\text{mask}} \}$
- **编辑集合** $\Delta_t = \{ i \mid x_t^i \neq v_t^i$ 且 $p_\theta(v_t^i \mid \bm{x}_t) > \tau_{\text{edit}} \}$

状态转移方程为：

$$
x_{t-1}^i = \begin{cases}
v_t^i & \text{if } i \in \Gamma_t \cup \Delta_t, \\
x_t^i & \text{otherwise}.
\end{cases}
$$

这一机制使得模型可以在生成过程中**实时回顾并修正自己的错误**，将线性的“生成”过程转变为迭代的“起草-编辑”过程。

### 3.2 双重运行模式：极速模式与高质量模式

基于可编辑框架，LLaDA2.1 提出了两种用户可配置的运行模式：

| 模式 | M2T 阈值策略 | T2T 角色 | 目标场景 |
|------|-------------|---------|---------|
| **极速模式 (S Mode)** | 激进降低 $\tau_{\text{mask}}$ | 承担大量纠错工作 | 追求极致吞吐，如代码生成 |
| **高质量模式 (Q Mode)** | 保守阈值 | 辅助性微调 | 追求推理精度，如数学证明 |

这种设计将僵化的速度-质量权衡转变为**灵活的、用户可控的连续谱**。

### 3.3 首个大规模 dLLM 强化学习框架 (EBPO)

LLaDA2.1 引入了 **ELBO-based Block-level Policy Optimization (EBPO)**，这是首个成功应用于大规模 dLLM 的强化学习框架。核心突破在于：

- 使用 ELBO 作为精确似然的代理，规避了序列级对数似然 $\log \pi_\theta(\bm{x})$ 的不可计算性。
- 通过**向量化似然估计（Vectorized Likelihood Estimation）**并行化边界计算，实现数量级加速。
- 采用分块因果掩码（Block-Causal Mask）确保每个块只能关注有效的历史上下文。

目标函数为裁剪的代理目标：

$$
\mathcal{J}_{\text{EBPO}}(\theta) = \mathbb{E}_{\bm{x}, \bm{y} \sim \pi_{\theta_{\text{old}}}} \left[ \min \left( \rho(\bm{y}|\bm{x}) \hat{A}, \text{clip}(\rho(\bm{y}|\bm{x}), 1-\epsilon_{\text{low}}, 1+\epsilon_{\text{high}}) \hat{A} \right) \right]
$$

### 3.4 多轮前向数据增强 (MTF) 与多块编辑机制 (MBE)

- **Multi-turn Forward (MTF)**：在 CPT 和 SFT 阶段，通过多轮前向传播暴露模型于更丰富的编辑场景，显著增强编辑能力。
- **Multi-Block Editing (MBE)**：超越单块编辑限制，允许模型基于新生成的块内容，回顾并修正先前已生成块中的错误，提升全局一致性。

## 4. 方法论

### 4.1 统一训练目标：M2T + T2T 混合损失

为了对齐“起草-编辑”推理范式，LLaDA2.1 在 CPT 和 SFT 阶段采用统一的双重流监督目标：

- **起草流 (Mask-to-Token)**：预测掩码位置的正确 token，建立基础的起草能力。
- **编辑流 (Token-to-Token)**：从随机噪声扰动中恢复原始 token，培养识别和重写错误的能力。

这种双重流训练确保模型在同一个参数空间内既是快速起草者，也是精确编辑者。

### 4.2 推理基础设施优化

LLaDA2.1 的推理基于定制化的 SGLang，并集成了多项工业级优化：

- **Alpha-MoE 大核**：将两个 FusedMoE 计算融合进单个 kernel，减少调度开销。
- **逐块 FP8 量化**：在推理速度和模型精度之间取得平衡。
- **块级因果掩码注意力**：允许长上下文的 KV Cache 在单次前向传播中完成计算。
- **Radix Caching 与批处理支持**：针对块扩散 LLM 的缓存和批处理优化。

## 5. 实验结果

论文在 33 个基准测试上进行了全面评估，涵盖知识、推理、代码、数学、Agent 与对齐五大维度。

### 5.1 质量对比 (Q Mode vs S Mode)

- 在 **Q Mode** 下，LLaDA2.1 在 Mini 和 Flash 两个尺寸上均**显著超越 LLaDA2.0**。
- 在 **S Mode** 下，指标相比 LLaDA2.0 略有下降，但换来了**推理吞吐量的巨大提升**。

### 5.2 速度表现 (S Mode 峰值 TPS)

| 模型 | HumanEval+ | BigCodeBench | LiveCodeBench |
|------|-----------|-------------|---------------|
| LLaDA2.1-Flash (100B, 量化后) | **891.74 TPS** | **801 TPS** | **663 TPS** |
| LLaDA2.1-Mini (16B, 量化后) | **1,586.93 TPS** | — | — |

> TPS = Tokens Per Second，每秒生成的 token 数。在代码领域表现尤为突出。

### 5.3 多块编辑 (MBE) 效果

在 S Mode 设置下引入 MBE 后：
- 推理和代码任务上性能提升最为显著。
- 吞吐量仅有**适度下降**，证明跨块迭代精炼能有效修正局部错误并改善全局一致性，而不会严重损害解码效率。

### 5.4 领域差异

实验观察到显著的领域差异：
- **代码、数学**等结构化领域：S Mode 可实现高速且精度损失极小。
- **通用对话、指令遵循**：对阈值更敏感，建议使用 Q Mode 或调参。

## 6. 结论与展望

### 6.1 主要结论

1. **编辑能力是加速的关键杠杆**：LLaDA2.1 证明，通过累积性错误修正，可以显著降低 dLLM 的解码阈值，从而在不牺牲过多质量的前提下获得巨大的推理速度收益。
2. **起草-编辑范式可行且高效**：在保持模型规模和训练数据几乎不变的情况下，仅通过改进解码方案即可实现“闪电般快速”的执行效率。
3. **速度-质量可配置**：用户可根据任务场景自由在极速和高质量之间切换，这种灵活性在实际部署中极具价值。

### 6.2 局限与未来方向

- **速度-精度权衡依然存在**：不同领域的最优阈值参数不同，通用场景下的自适应阈值选择仍需研究。
- **编辑能力尚处早期**：dLLM 的编辑能力研究刚刚起步，未来如将编辑机制更深地融入强化学习（如编辑感知的奖励函数），有望进一步提升性能。
- **边缘情况**：过度激进的阈值可能产生诸如 n-gram 重复的“口吃”瑕疵，需要在起草速度和初始结构质量之间寻找最佳操作边界。
- **实验阶段**：LLaDA2.1 仍处于实验阶段，某些极端场景可能出现意外行为。

## 7. 个人评价

### 7.1 优点

1. **范式创新而非规模竞赛**：在业界普遍追求“大力出奇迹”的Scaling Law背景下，LLaDA2.1选择了一条更具巧思的路径——通过引入“可编辑性”这一结构性创新，在不增加模型参数和训练数据规模的前提下，实现了推理速度的数量级提升。这种对模型能力的重新定义（从“只能生成”到“既能生成也能编辑”）体现了深刻的系统级思考。

2. **工程与理论并重**：论文不仅有扎实的数学形式化（双重阈值、EBPO目标函数、状态演化方程），更有工业级的系统实现（Alpha-MoE、FP8量化、SGLang定制化、批处理支持）。从CPT到SFT再到RL的完整训练管线设计，显示出这是一个经过充分验证的production-ready方案。

3. **辩证看待并行解码**：论文坦诚指出dLLM的并行解码并非免费的午餐——高并行度在带来理论速度优势的同时，也引入了比AR模型更高的错误率。这种对问题本质的洞察力，是推动“编辑机制”成为核心解的关键。

4. **用户价值导向**：S Mode和Q Mode的设计不是简单的技术炫技，而是真正考虑到了用户在不同场景下的实际需求。代码生成需要速度，数学推理需要精度，这种场景化的可配置性极具工程实用价值。

### 7.2 不足

1. **阈值选择的“黑魔法”**：论文虽然定义了$\tau_{\text{mask}}$和$\tau_{\text{edit}}$，但对于如何根据任务自动选择最优阈值对，缺乏系统性的指导原则或自动化方法。不同领域的“调参”似乎仍依赖启发式经验。

2. **S Mode下的质量下降**：虽然论文强调了S Mode的速度优势，但相比Q Mode和LLaDA2.0的质量下降是不可忽视的。对于需要高质量输出的场景，这种权衡是否可接受仍有待商榷。

3. **编辑机制的理论边界**：论文尚未深入探讨编辑能力的理论上限——例如，模型能否识别并修正“结构性错误”（如逻辑矛盾、事实错误），而不仅仅是“拼写/语法错误”？目前的T2T编辑更像是局部token替换，距离真正的“深层重写”尚有距离。

### 7.3 总体评价

LLaDA2.1 是一篇具有里程碑意义的论文。它成功地将 dLLM 从“生成式范式”推向“生成-编辑混合范式”，为离散扩散语言模型的实用化部署开辟了新的可能性。892 TPS 的代码生成速度对于 100B 模型而言堪称惊艳，足以在实际生产环境中与自回归模型一较高下。尽管可编辑 dLLM 仍处于早期阶段，但其展现出的潜力令人振奋——未来若能将编辑机制与强化学习更深地结合，或将催生出真正意义上的“自纠错智能体”。
